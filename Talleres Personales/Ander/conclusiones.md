# Bases para el desarrollo web

## Resumen

Al iniciar en el mundo del desarrollo web existen ciertas tecnologías que son imprescindibles, la base sólida para el desarrollo web, estas son HTML, Css y JavaScript, además del control de bases de datos y sus lenguajes. Cada una de ellas abarca una arquitectura relacionada la una con la otra.

Toda página o aplicación web esta compuesta por las mismas por lo que con el paso del tiempo se crearon diferentes herramientas para explotar las capacidades que ejercen estas, las cuales funcionan como entorno de trabajo con sus propios estándares que en conjunto unos con otros serian clave para la creación de grandes proyectos.

Cualquier persona o empresa que necesite un software web también debería conocer como se crean los mismos  por lo que se pretende informar de manera detallada todo acerca de cómo están compuesta cualquier aplicación web y su paso a paso en el desarrollo general un proyecto.

## Abstract

At the beginning in the world of web development there are certain technologies that are essential, the solid foundation for web development, these are HTML, Css and JavaScript in addition to database control and its languages. Each of them encompasses an architecture related to each other. Every page or web application is made up of them, so over time different tools were created to exploit the capabilities they exercise, which function as a work environment with their own standards that, together with each other, would be key to the creation of great projects.

Any person or company that needs web software should also know how they are created, so it is intended to inform in detail everything about how any web application is composed and its step by step in the general development of a project.

Web design encompasses all aspects of creating Web sites, from the structure of Web pages and the markup which controls it, through scripts that add interactivity and generate pages dynamically, to issues of accessibility, usability and visual communication[1].

## I. INTRODUCCIÓN

En un poco más de una década desde su aparición, la World Wide Web  se ha convertido en un instrumento de uso cotidiano en nuestra  sociedad, comparable a otros medios tan importantes como la radio, la televisión o el teléfono, a lo que aventaja en múltiples aspectos [2].

Hoy en día existen incontables números de paginas o aplicaciones web con diferentes propósitos en general, normalmente se interactúa con alguna en el día a día , pero poco se sabe acerca de como están construidas, que tecnologías usan e incluso si son seguras y, con excepción a lo ultimo esta bien no tener conocimiento acerca de toda la infraestructura de una aplicación, pero, si se desea crear un proyecto este tipo es altamente necesario saber como será construido para así tener un mejor panorama y seguridad de llevar a cabo un proceso seguro y de calidad.

Para el correcto funcionamiento, éxito y rentabilidad en base al desarrollo de una aplicación web es necesario hacer uso de diferentes tecnologías, tanto para el Back End que se podría definir como la lógica detrás del proyecto y Front End como interfaz gráfica o todo lo que se puede ver y con lo que se puede interactuar . Anteriormente se podían crear páginas web desde el backend hasta el frontend sin mucha complicación, pero hoy en día la tecnología ha avanzado al punto que no existe una herramienta que por sí misma provea lo necesario para un desarrollo escalable y profesional , por lo que empezaron a surgir diversas tecnologías que en conjunto e interactuando una con la otra generarían grandes posibilidades para diversos objetivos.

Además de esto, es importante resaltar el tipo de metodologías que existen a la hora de llevar a cabo un proyecto y con ellas el ciclo de vida de una aplicación o programa.

## II. DESARROLLO DEL TEMA

El desarrollo web tiene como definición construir y mantener aplicaciones o sitios que funcionen en internet, permite que se tenga una apariencia impecable y un excelente funcionamiento, junto con la mejor experiencia de usuario . El desarrollo web funciona a través de diferentes tecnologías y lenguajes de programación destinados a diferentes áreas. El desarrollo de aplicaciones web se conforma de la capa de presentación (FrontEnd) y una capa de acceso a datos (BackEnd) [3], se divide de forma general en dos partes esta sería la arquitectura Cliente – Servidor que es en la que se basa la web. El Cliente es quien pide la información y el Servidor la envía, siempre que el Cliente tenga los permisos necesarios.

### A. Frontend

Se refiere a la parte que interactúa con el usuario, todo lo que se puede ver cuando se accede a una pagina o aplicación web (Botones, imágenes, textos, inputs) todos los elementos con los que se puede interactuar se consideran parte del Frontend, principalmente están compuestos por un conjunto de tecnologías consideradas la base para el desarrollo web, estás son: HTML, CSS y JavaScript.

#### 1) HTML

Hyper Text Markup Language es el fundamento de la World Wide Web, como lenguaje de marcado o etiquetas para desarrollar páginas web, no es un lenguaje de programación por lo que su aprendizaje suele ser más rápido y fluido. HTML puede describirse como el esqueleto de una página web.

#### 2) CSS

Cascade Style Sheet Es un lenguaje de diseño gráfico que se usa para establecer el diseño visual de las páginas o aplicaciones web, con este se define la apariencia de la web (colores, tamaños, formas, animaciones, etc).

#### 3) JavaScript

Es un lenguaje de programación orientado a objeto que funciona en la web de manera nativa, esto quiere decir que no necesita ser compilado para su uso siempre que se ejecute en la web. Del lado del cliente o Frontend con este lenguaje se pueden crear diferentes efectos y animaciones como interacción, pero también puede ser utilizado del lado del servidor implementando tecnologías especializadas como Ajax o NodeJs. Un buen desarrollador web tiene completamente dominadas estas herramientas, pues son la base para alcanzar casi cualquier tipo de proyecto.

¿Como se relacionan estas tecnologías al desarrollar una página o aplicación web?

Primeramente, se empezaría por la estructura del sitio según se necesite, utilizando HTML esto se puede ejemplificar como la construcción de una casa, está hecha con todos sus espacios y habitaciones, aunque sin ventanas, puertas, pintura u algo dentro. Después se empezaría con el CSS como el diseño o apariencia que se le quiere dar a la casa, su color, ventanas, puertas y decoraciones. Por último, se añadiría JavaScript como cualquier tipo de cosa con la que se pueda interactuar, como los muebles, relojes, televisores, etc. Hasta este punto por parte del frontend estaría construida una pagina web funcional, una estructura, un diseño y una interacción.

### B. Backend

Dentro de la programación el Backend comprende toda la lógica y programación detrás de un proyecto. Puede recibir y enviar información según se necesite, es una parte fundamental en el desarrollo de software ya que puede generar una mejor o peor experiencia de usuario, así como un fuerte o débil nivel de seguridad. Las tecnologías más utilizadas en la actualidad son NodeJs, Python, Java, Php, entre otros. El desarrollador BackEnd debe de conocer de bases de datos, frameworks y aspectos de seguridad [4].

El backend es tan importante como el frontend en un proyecto de desarrollo web, pues esto aumenta la interacción que se tiene en la pagina o aplicación, al reunir y enviar información de forma instantánea, para así desencadenar cualquier tipo de acción que se refleje en la aplicación. Usando como ejemplo YouTube, al buscar algo se envía una información al Backend que es el responsable de traer una lista con cada video relacionado a lo que se escribe en el buscador, posteriormente al elegir alguno este traerá consigo el número de Me gusta, No me gusta y comentarios.

A continuación se presentan las tecnologías más utilizadas en el desarrollo de Backend:

#### 1) NodeJs

Es un entorno de ejecución en tiempo real basado en JavaScript que utiliza un modelo de entrada y salida de datos flexible, esto puede referirse a cualquier operación, desde leer o escribir cualquier tipo de archivos hasta hacer algún tipo de solicitud, siendo liviano y eficiente frente a otros [5].

#### 2) Java

Java es probablemente el lenguaje de programación backend más influyente en todo el mundo teniendo su estabilidad como punto más fuerte. Es completamente orientado a objetos por lo que cualquier persona con conocimientos en este tipo de programación debería ser capaz de controlarlo fácilmente.

#### 2) Python

Es uno de los lenguajes de programación multipropósito líderes en la actualidad, es muy relevante en la industria del software ya que puede llevar a cabo cualquier tipo de proyecto tecnológico. Es fácil entender este lenguaje de programación pues tiene un estilo similar al idioma inglés por lo que es altamente legible para cualquier persona. Tiene una enorme cantidad de librerías por lo que reduce la necesidad de escribir código manual.

#### 3) Php

Es un lenguaje de script, programación multipropósito que es fácil de usar, se enfoca principalmente en la comunicación del lado del servidor por lo que puede ejercer funciones como recopilación de datos de formularios, administrar archivos en el servidor, modificar bases de datos, etc. Es más utilizado para el desarrollo web por su capacidad de incrustarse en archivos HTML.

### C. Frameworks y Librerias

Los frameworks nacen de la necesidad de estructurar y normalizar el código del sistema, facilitando un patrón o esqueleto para el desarrollo y la implementación de aplicaciones, con esto podemos decir que un framework es una especie de plantilla o esquema predeterminado que sirve como punto de partida para la organización y el desarrollo de un software. Facilita en gran medida el desarrollo.

El uso de frameworks para cualquier tipo de desarrollo reduce el tiempo de elaboración e implementación y ayuda a hacer un trabajo mantenible y escalable, según las características del mismo [5].

A continuación se presentan los frameworks más utilizados en el desarrollo de Backend y Frontend:

#### 1) Angular

Angular es una plataforma de desarrollo, construida en TypeScript. Como plataforma, Angular incluye:

Un marco basado en componentes para crear aplicaciones web escalables, una colección de bibliotecas bien integradas que cubren una amplia variedad de funciones, incluido el enrutamiento, la gestión de formularios, la comunicación cliente-servidor y más, un conjunto de herramientas de desarrollo para ayudarlo a desarrollar, compilar, probar y actualizar su código.


#### 2) React

Es una librería basada en componentes React ayuda a crear interfaces de usuario interactivas de forma sencilla. Diseña vistas simples para cada estado en tu aplicación, y React se encargará de actualizar y renderizar de manera eficiente los componentes correctos cuando los datos cambien. Las vistas declarativas hacen que tu código sea más predecible, por lo tanto, fácil de depurar.


#### 3) Django

Django es un framework web de alto nivel que permite el desarrollo rápido de sitios web seguros y mantenibles. Desarrollado por programadores experimentados, Django se encarga de gran parte de las complicaciones del desarrollo web, por lo que puedes concentrarte en escribir tu aplicación sin necesidad de reinventar la rueda. Es gratuito y de código abierto, tiene una comunidad próspera y activa, una gran documentación y muchas opciones de soporte gratuito y de pago.


#### 4) Express NodeJs

Express es el framework web más popular de Node, y es la librería subyacente para un gran número de otros frameworks web de Node populares. A pesar de que Express es en sí mismo bastante minimalista, los desarrolladores han creado paquetes de middleware compatibles para abordar casi cualquier problema de desarrollo web. Hay librerías para trabajar con cookies, sesiones, inicios de sesión de usuario, parámetros URL, datos POST, cabeceras de seguridad y muchos más.

En un nivel más avanzado la producción de páginas o aplicaciones web profesionales esta alineada a un orden especifico utilizando diferentes tipos de frameworks no solo por su utilidad a la hora de reducir el tiempo de trabajo, si no por su orden, gracias a esto se crea un parámetro de desarrollo para hacer más fácil de mantener, pues según el framework o libreria sin importar lo diferente que sea un proyecto de otro va a estar compuesto en un orden fácil de entender, para buscar donde se encuentran los componentes. Para el desarrollo se adoptó un proceso secuencial con orientación Web; por su estructura, en este tipo de proyectos se puede sacar provecho a su organización. El proceso genérico del marco de trabajo comunicación, planeación, modelado, construcción y despliegue es aplicable a la web [6].

## III. METODOLOGÍA 

Al desarrollar software en general se debe planificar y estructurar el proyecto, para esto se debe tener en cuenta los requerimientos del cliente, los cuales se deben analizar y definir para poder desarrollar el software. En el mundo de desarrollo web, el proceso de desarrollo en ocaciones puede ser muy competitivo y para poder tener un buen producto en poco tiempo se deben aplicar metodologías de desarrollo agiles que minimicen los tiempos de desarrollo y poco a poco puedan tener prototipos funcionales antes de que este listo el producto en su totalidad.

Algunas de estas metodologías son:

#### 1) Scrum

Es una metodología ágil para la gestión y planificación de proyectos de desarrollo de software. Se basa en la idea de que el trabajo de un equipo de desarrollo de software se puede dividir en tareas pequeñas y que se pueden organizar en ciclos de tiempo cortos, llamados sprints. En cada sprint, el equipo se compromete a completar un conjunto de tareas definidas como "completado" y a entregar un producto incrementable, que es una versión funcional del software, al final del sprint. El objetivo de cada sprint es crear un producto potencialmente entregable de software que cumpla con los requisitos de los clientes y que pueda probarse. Los sprints tienen una duración de entre una y cuatro semanas y se llevan a cabo de forma repetida hasta que el producto esté completo. [7]

#### 2) Kanban

Kanban es un sistema de gestión de proyectos que se basa en la mejora continua, la visualización del trabajo en curso y la colaboración entre los equipos. Kanban se centra en la maximización de la eficiencia y la satisfacción del cliente mediante la entrega oportuna de valor. Kanban se basa en la idea de que el trabajo de un equipo de desarrollo de software se puede dividir en tareas pequeñas y que se pueden organizar en ciclos de tiempo cortos, llamados sprints. En cada sprint, el equipo se compromete a completar un conjunto de tareas definidas como "completado" y a entregar un producto incrementable, que es una versión funcional del software, al final del sprint. El objetivo de cada sprint es crear un producto potencialmente entregable de software que cumpla con los requisitos de los clientes y que pueda probarse. Los sprints tienen una duración de entre una y cuatro semanas y se llevan a cabo de forma repetida hasta que el producto esté completo. [8]

## IV. RESULTADOS

Ahora con un panorama mas general acerca de las bases para el desarrollo web, nos damos cuenta que para construir un producto profesional con tecnologias web debemos tener encuenta la metodologia, las tecnologias de frontend y backend, y las librerias que nos ayudaran a desarrollar el producto de una manera mas eficiente y rapida.

En ejemplo que nos puede dar buenos resultados es hacer una web en Angular como framework de frontend y Express como framework de backend, y para el manejo de la base de datos se puede usar MongoDB como base de datos no relacional, y una metodologia como Scrum para el desarrollo del producto, asi garantizamos que el marco de trabajo de desarrollo web se cumpla y el producto sea de calidad en poco tiempo.

Algunas de las similitudes que tienen estos dos frameworks son:

React es sencillo, se puede empezar a aprender y crear un proyecto en mucho menos tiempo que otras plataformas. Además, React es una plataforma muy flexible y fácil de aprender. Por lo tanto, es una buena opción para los desarrolladores que están empezando a aprender a programar.

## VI. CONCLUSIONES

### Frontend

Angular y React son dos de los frameworks de frontend mas populares y utilizados en la actualidad. A la hora de tomar una desicion sobre cual utilizar, debemos tener en cuenta las necesidades de nuestro proyecto, ya que cada uno tiene sus caracteristicas que los hacen mas o menos adecuados para un proyecto en particular.

Por ejemplo en una gran empresa con personal capacitado, Angular es una buena opcion ya que es muy eficaz en el desarrollo de aplicaciones multiplataforma, por lo que se puede reducir los gastos. Pero al mismo tiempo, Angular es una plataforma muy compleja, por lo que requiere un elevado esfuerzo y experiencia para dominarla.

Por otro lado, React es una plataforma muy flexible y fácil de aprender, por lo que es una buena opcion para los desarrolladores que están empezando a aprender a programar. Ademas, se usa con JasaScript puro, ademas el JSX permite combinar HTML y JavaScript en un mismo archivo. Esto hace que el código sea más legible y fácil de entender.

### Backend

Django y Node.js son dos de los frameworks de backend mas populares y utilizados en la actualidad.

Django es un framework ded python enfocado en el desarrollo web, mientras que Node.js es un entorno de ejecución de JavaScript multiproposito, con ayuda de Express se puede crear un servidor web con Node.js con características muy similares a las de Django.

Ultimamente Node.js ha ganado mucha popularidad, pero Django sigue siendo una buena opcion para proyectos web, ya que es un framework muy completo y robusto, con una gran comunidad de desarrolladores que enriquecen regularmente los recursos, lo que hace que sea mas facil encontrar ayuda en caso de tener algun problema.
